## 一、 研究背景與問題定義

現代人面對高壓環境，心理支持系統（Psychological Support Systems）的需求日益漸增。然而，傳統的心理諮商具備較高的門檻，而市面上的情緒抒發 App 則多偏向單向記錄（如心情日記），缺乏即時且具備同理心的雙向互動。

本專題旨在開發一套「基於敘事療癒（Narrative Therapy）與情緒投射（Emotional Projection）之互動式故事館系統」。我們利用大眾熟知的經典文本作為安全網（Safety Net），引導使用者將個體經驗與負面情緒投射於故事環節中。本系統的核心技術挑戰在於：**如何透過多模態 AI 的有機整合，實時生成具備高度同理心（Empathy-driven）的文本回應、動態情境視覺與高表現力的語音，引導使用者完成「情緒宣洩（Catharsis）— 溫柔承接 — 導回主線」的完整心理閉環。

## 二、 核心技術瓶頸：為何「免費版 API」無法滿足學術與應用需求？

在前期可行性評估（Feasibility Study）中，我們使用免費版或開源開箱即用模型進行了原型測試，發現其效能在本研究情境下存在結構性缺陷：

1. 語意理解深度與情商（EQ）不足（LLM 瓶頸）：

   免費或低階模型（如 GPT-3.5）在面對複雜、細膩且可能帶有潛在負面情緒的文本時，傾向給出說教式、罐頭化（Canned Responses）的回答，嚴重破壞敘事療癒所需的「無條件積極關懷（Unconditional Positive Regard）」原則。

2. 多模態生成缺乏上下文一致性（Context Consistency 缺陷）：

   在視覺與聽覺生成上，免費工具無法有效鎖定種子碼（Seed）與風格特徵。當使用者進行下一輪互動時，角色外貌崩壞、場景風格突變、語音變為生硬的機械音。這種「出戲」的體驗會破壞使用者的投射效應，使專題失去實用價值。

因此，引入具備高度上下文控制力與高參數密度的付費版 API，是確保本專題具備應用研究價值的必要條件。

## 三、 技術架構規劃與付費 API 必要性論證

本系統預計整合五大核心多模態模組，其技術選型與經費投入之必要性論證如下：

```

[使用者輸入: 情緒文本] 

       │

       ▼

[核心控制大腦: Claude 3.5 / GPT-4o] ───► 分析情緒、生成「溫柔承接」文本

       │

       ├────────────────────────┼────────────────────────┐

       ▼                        ▼                        ▼

[視覺生成: Midjourney]   [語音生成: ElevenLabs]   [音訊生成: Suno/Udio]

(角色一致性與場景控制)   (具備情感語調的說書人)   (動態情緒氛圍配樂)


```

### 1. 核心控制大腦：相應回覆與情境導引 (LLM)

 技術選型：Anthropic Claude 3.5 Sonnet 或 OpenAI GPT-4o

 論證依據：根據目前主流的 LLM Benchmark，Claude 3.5 與 GPT-4o 在長文本脈絡理解（Context Window）與情商表現上顯著優於其他模型。我們需要其強大的 Prompt Engineering 容納能力，在維持系統設定（System Prompt：說書人角色、溫柔承接語氣）的同時，能精準解析使用者輸入的語意特徵，並將該特徵動態編織進故事的下一階段主線。

### 2. 聽覺靈魂：具備情感表現力的說書人語音 (TTS)

 技術選型： ElevenLabs 或 OpenAI TTS (HD 模式)

 論證依據：互動故事的沉浸感高度依賴語音的質地。付費 API 提供的高擬真音訊（High-Fidelity Audio）支援微調語速、停頓與情感色調（如：溫暖、低沈、安慰）。這能建立數位心理支持所需的「陪伴感」，非免費版之語音合成技術可比擬。

### 3. 視覺一致性控制：場景與動態角色生成 (Text-to-Image)

 技術選型：Midjourney API (或基於 ComfyUI 部署之 Stable Diffusion 付費算力)

 論證依據：本專題涉及同一角色、不同場景的連續敘事。我們需要利用付費服務所支援的**角色權重參考（Character Reference, --cref）與風格權重參考（Style Reference, --sref）功能。唯有確保角色在悲傷、釋懷等不同情緒狀態下「外貌一致，僅微調表情與肢體」，方能滿足心理學上對特定客體（Object）的投射需求。

4. 氛圍渲染器：情境驅動背景音樂 (Music Generation)

 技術選型：Suno AI 或 Udio API

 論證依據：為了配合故事走向與使用者當下的情緒轉折，背景音樂必須是動態且客製化的。付費 API 允許系統根據 LLM 輸出的「情緒標籤（Emotion Tags）」，即時生成結構完整、無版權爭議且融合度高的環境配樂，強化沉浸體驗。

四、 預期研究產出與效益评估

1. 多模態 AIGC 系統整合架構（System Integration）： 實證如何將文本、影像、語音、音樂四大相異維度的生成式 AI 技術，透過 API 進行異質整合與即時串接。
2. 精準 Prompt 工程實踐：探索大語言模型在心理諮商初談與敘事導引上的應用邊界，並產出一套具備高包容度、非批判性對話的 Prompts 資產。
3. 具備商品化潛力的 Prototype：產出一個高完整度的 Web 或 App 原型，未來可延伸至校園心理諮商輔助工具或大眾舒壓應用。
